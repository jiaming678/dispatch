# Markdown
In addition to the text component, the message card also provides the Markdown component. Markdown component can be used to render text formats, pictures, dividing lines and other elements. This article will introduce Markdown component and corresponding JSON parameter descriptions.
This document refers to legacy message cards and is no longer maintained. To view the corresponding updated documentation, refer to [Rich text](/document/uAjLw4CM/ukzMukzMukzM/feishu-cards/card-components/content-components/rich-text).
## Component display
In the message card building tool, the Markdown component is shown in the figure below. After adding the component, you can adjust configuration items such as \*\*Style\*\*, \*\*Content\*\*, and \*\*Interaction\*\*.
The message card building tool currently does not support debugging and preview the ordered list, unordered list, and code block syntax of the Markdown component.
![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/7da4c4eaa2537430abfbd7973ed8d567\_3Ke7qUmazf.png?height=1414&lazyload=true&maxWidth=600&width=2882)
In addition, the text component provided by the tool supports adjusting the text format to Markdown. The corresponding JSON description is to set `"tag": "lark\_md"` in the `text` element. For details, see [text component](/document/ukTMukTMukTM/uUzNwUjL1cDM14SN3ATN).
\* Markdown tags that are independent of text formatting (e.g., images, dividing lines) cannot be used in text elements.
\* This article only introduces the Markdown content format of the card. When actually sending the card message, you need to combine it with the parameter configuration of the specific interface. For example, when calling the [Send Message](/document/uAjLw4CM/ukTMukTMukTM/reference/im-v1/message/create) interface, the Markdown card content needs to be passed into the `content` field of the interface. In addition, it needs to be set according to the card content. Message type (`msg\_type`), message recipient ID (`receive\_id`) and other fields.
## Parameter description
The parameters contained in the Markdown component are described in the following table.
| Parameters | Required | Type | Description |
| --- | --- | --- | --- |
| tag | Yes | String | Markdown component label, fixed value: markdown. |
| content | Yes | String | Construct Markdown content using supported Markdown syntax. Syntax details can be found in the \*\*Supported Syntax\*\* section below. |
| text\\_align | No | String | Set the alignment of text content. Value: \* \*\*left\*\*: Left aligned \* \*\*center\*\*: center alignment \* \*\*right\*\*: right aligned \*\*Default\*\*: left |
| href | No | Object | Differential jump. Only use it when you need to jump to different links on PC or mobile. Example values: ``` { "urlVal": { "url": "https://feishu.com", "android\_url": "https://android.com/", "ios\_url": "https://apple.com/", "pc\_url": "https://windows.com" } } ``` |

## Card example
### Markdown component
An example of usage in the message card Markdown component is shown in the following JSON code.
```
{
"elements":[
{
"tag":"markdown",
"href":{
"urlVal":{
"url":"xxx1",
"pc\_url":"xxx2",
"ios\_url":"xxx3",
"android\_url":"xxx4"
}
},
"content":"Normal text\nStandard emoji😁😢🌞💼🏆❌✅\n\*italic\*\n\*\*bold\*\*\n~~strikethrough~~\n[text link](www.example. com)\n[Differential jump]($urlVal)\n"
},
{
"tag":"hr"
},
{
"tag":"markdown",
"content":"The above is a dividing line\n![hover\_text](img\_v2\_16d4ea4f-6cd5-48fa-97fd-25c8d4e79b0g)\nThe above is a picture label"
}
],
"header":{
"template":"blue",
"title":{
"content":"This is the card title bar",
"tag":"plain\_text"
}
}
}
```

### lark\\_md mode for text
By declaring the `tag` field of [text component](/document/ukTMukTMukTM/uUzNwUjL1cDM14SN3ATN) as `lark\_md`, you can use Markdown tags to configure the text format in `text`. The usage example is as follows JSON code.
```
{
"elements":[
{
"tag":"div",
"text":{
"tag":"plain\_text",
"content":"text-lark\_md",
"lines":1
},
"fields":[
{
"is\_short":false,
"text":{
"tag":"lark\_md",
"content":"https://open.feishu.cn"
}
},
{
"is\_short":false,
"text":{
"tag":"lark\_md",
"content":"ready\nnew line"
}
},
{
"is\_short":false,
"text":{
"tag":"lark\_md",
"content":"\*Italic\*"
}
},
{
"is\_short":false,
"text":{
"tag":"lark\_md",
"content":"\*\*Bold\*\*"
}
},
{
"is\_short":false,
"text":{
"tag":"lark\_md",
"content":"~~delete line~~"
}
},
{
"is\_short":false,
"text":{
"tag":"lark\_md",
"content":""
}
}
]
}
]
}
```
The effect is shown in the figure below:
